从 Prompt 到架构方案

AI 协作不是一句话让它写代码，而是先让它理解需求、提出方案、比较取舍，再进入实现和验证。

AI 工程能力Prompt to ArchitectureTypeScript + Python方案生成

今日目标

  • 学会让 AI 先分析而不是直接实现。
  • 要求 AI 给多个方案并比较取舍。
  • 把方案落到目录、接口、测试和 ADR。

1. 这个方法解决什么问题

直接让 AI “实现某功能”，它通常会按局部最短路径写代码。架构方案需要先明确目标、约束、风险和演进方向。

2. 一个生活类比

盖房前不能直接叫工人开工，要先看地形、预算、用途、工期，再出方案。

3. 适用场景

  • 新增模块。
  • 重构旧系统。
  • 接入外部系统。
  • 需要比较多种架构方案。

4. 不适用场景

  • 小修小补。
  • 需求还完全不清楚。

5. 核心结构

步骤要求 AI 输出
理解需求目标、范围、未知点。
给方案2-3 个候选方案。
比较取舍复杂度、风险、演进成本。
落地设计目录、接口、数据流、测试。
记录决策ADR 草案。

6. 示例代码

TypeScript 示例

type ArchitectureOption = { name: string; complexity: "low" | "medium" | "high"; risks: string[] };
const option: ArchitectureOption = {
  name: "模块化单体",
  complexity: "medium",
  risks: ["需要约束模块依赖"],
};

Python 示例

option = {
    "name": "模块化单体",
    "complexity": "medium",
    "risks": ["需要约束模块依赖"],
}

7. 优缺点

收益代价
减少 AI 直接写错方向。前期多一步分析。
方案可比较、可记录。需要人做最终取舍。

8. 和普通 Prompt 的区别

普通 Prompt架构 Prompt
帮我实现订单模块。先分析订单模块边界，给 3 个方案并比较，再生成推荐方案。

9. 一个练习题

写一个 Prompt，让 AI 为“通知系统”提出架构方案，不要直接写代码。

10. 参考答案或改进建议

要求 AI 输出目标、约束、候选方案、推荐方案、模块边界、失败处理、测试策略和 ADR 草案。最后由人确认后再实现。

今日总结

好的 Prompt 不是命令 AI 写代码，而是引导 AI 完成工程设计流程。先方案，后代码。
