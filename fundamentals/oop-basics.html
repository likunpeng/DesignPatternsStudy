面向对象基础学习笔记

面向对象的五个核心概念——封装、继承、多态、组合、接口——是所有设计模式的基石。掌握它们之间的区别和选择原则，能让你在设计模式的学习中事半功倍。

今日目标

  • 理解封装、继承、多态、组合、接口的核心含义。
  • 能用 TypeScript 和 Python 写出每种概念的示例。

  • 理解为什么接口是设计模式中最重要的概念之一。

1. 封装

核心思想：把数据和操作封装在对象内部，外部只能通过公开方法访问，避免直接操作内部状态。

生活类比： ATM 机只暴露插卡口和出钞口，内部电路和线缆被外壳封住。你不需要知道钱是怎么从钱箱里数出来的。

坏代码

class BankAccount {
  balance = 0;
}

const account = new BankAccount();
account.balance = -99999;  // 直接改，没有任何校验

TypeScript 示例

class BankAccount {
  private _balance = 0;

  get balance(): number {
    return this._balance;
  }

  deposit(amount: number): void {
    if (amount <= 0) throw new Error("金额必须大于 0");
    this._balance += amount;
  }

  withdraw(amount: number): void {
    if (amount <= 0) throw new Error("金额必须大于 0");
    if (amount > this._balance) throw new Error("余额不足");
    this._balance -= amount;
  }
}

Python 示例

class BankAccount:
    def __init__(self):
        self._balance = 0  # 前置下划线约定为"内部使用"

    @property
    def balance(self) -> int:
        return self._balance

    def deposit(self, amount: int) -> None:
        if amount <= 0:
            raise ValueError("金额必须大于 0")
        self._balance += amount

    def withdraw(self, amount: int) -> None:
        if amount <= 0:
            raise ValueError("金额必须大于 0")
        if amount > self._balance:
            raise ValueError("余额不足")
        self._balance -= amount
语言差异：TypeScript 有 private 关键字强制限制访问；Python 依靠命名约定（_field）和 @property，由开发者自觉遵守。

2. 继承

核心思想：子类可以从父类获取属性和方法，实现代码复用，表达"是一种"的关系。

生活类比：电动车和燃油车都"是一种"车，都有方向盘、刹车、加速踏板，但具体动力来源不同。

坏代码

class Car {
  start() { /* ... */ }
  stop() { /* ... */ }
}
class ElectricCar {
  start() { /* 重复 Car 的逻辑 */ }
  stop() { /* 重复 Car 的逻辑 */ }
  charge() { /* ... */ }
}

TypeScript 示例

class Car {
  brand: string;
  constructor(brand: string) { this.brand = brand; }

  start(): void { console.log(`${this.brand} 启动`); }
  stop(): void { console.log(`${this.brand} 熄火`); }
}

class ElectricCar extends Car {
  charge(): void { console.log(`${this.brand} 充电中`); }
}

const tesla = new ElectricCar("特斯拉");
tesla.start();   // 继承来的
tesla.charge();  // 自己的

Python 示例

class Car:
    def __init__(self, brand: str):
        self.brand = brand

    def start(self) -> None:
        print(f"{self.brand} 启动")

    def stop(self) -> None:
        print(f"{self.brand} 熄火")


class ElectricCar(Car):
    def charge(self) -> None:
        print(f"{self.brand} 充电中")


tesla = ElectricCar("特斯拉")
tesla.start()
语言差异：两者写法接近。Python 支持多重继承，TypeScript 只能单继承类但可以多实现接口。

3. 多态

核心思想：同一个方法名，不同对象可以有不同的实现方式。调用方只关心"能做这个动作"，不关心"具体怎么做"。

生活类比：老师对学生说"开始考试"，每个学生答题方式不同，但老师不需要知道每个人具体怎么做。

坏代码

function playSound(animal: any) {
  if (animal.type === "dog") console.log("汪汪");
  else if (animal.type === "cat") console.log("喵喵");
  else if (animal.type === "duck") console.log("嘎嘎");
}

TypeScript 示例

interface Animal {
  speak(): string;
}

class Dog implements Animal {
  speak(): string { return "汪汪"; }
}

class Cat implements Animal {
  speak(): string { return "喵喵"; }
}

function makeItSpeak(animal: Animal): void {
  console.log(animal.speak());
}

makeItSpeak(new Dog());  // 汪汪
makeItSpeak(new Cat());  // 喵喵

Python 示例

from typing import Protocol


class Animal(Protocol):
    def speak(self) -> str:
        ...


class Dog:
    def speak(self) -> str:
        return "汪汪"


class Cat:
    def speak(self) -> str:
        return "喵喵"


def make_it_speak(animal: Animal) -> None:
    print(animal.speak())


make_it_speak(Dog())
make_it_speak(Cat())
语言差异：TypeScript 多态依赖接口或继承；Python 依赖鸭子类型——只要对象有 speak 方法就能用，不需要显式声明接口。

4. 组合

核心思想：一个类持有另一个类的实例作为成员，而不是通过继承获得能力。表达"有一个"的关系。

生活类比：汽车"有一个"引擎，而不是"是一台"引擎。引擎坏了可以换，但如果汽车继承了引擎，换引擎就等于换车。

坏代码

class Engine { start() { /* ... */ } }

class Car extends Engine {  // Car 是 Engine？不合理！
  drive() { this.start(); }
}

TypeScript 示例

class Engine {
  start(): void { console.log("引擎启动"); }
  stop(): void { console.log("引擎熄火"); }
}

class Car {
  constructor(private engine: Engine) {}

  drive(): void {
    this.engine.start();
    console.log("行驶中");
    this.engine.stop();
  }
}

const car = new Car(new Engine());
car.drive();

Python 示例

class Engine:
    def start(self) -> None:
        print("引擎启动")

    def stop(self) -> None:
        print("引擎熄火")


class Car:
    def __init__(self, engine: Engine):
        self.engine = engine

    def drive(self) -> None:
        self.engine.start()
        print("行驶中")
        self.engine.stop()


car = Car(Engine())
car.drive()
关键判断：如果两个类的关系是"A 是 B"，用继承；如果是"A 用到 B"，用组合。大多数情况下组合比继承更灵活。

5. 接口

核心思想：定义一组方法签名，不提供实现。任何类只要符合这个契约，就能被调用方使用。接口让系统"面向契约编程"。

生活类比： USB 接口定义了"能插进去、能传数据"的规范，U 盘、鼠标、键盘都实现了这个接口，电脑不关心具体是什么设备。

坏代码

function saveData(storage: any) {
  if (storage instanceof MySQL) storage.insert(data);
  else if (storage instanceof Mongo) storage.create(data);
  else if (storage instanceof Redis) storage.set(data);
}

TypeScript 示例

interface DataStorage {
  save(data: string): void;
}

class MySQLStorage implements DataStorage {
  save(data: string): void { console.log(`MySQL 存储: ${data}`); }
}

class MongoStorage implements DataStorage {
  save(data: string): void { console.log(`Mongo 存储: ${data}`); }
}

function writeData(storage: DataStorage, data: string): void {
  storage.save(data);
}

writeData(new MySQLStorage(), "用户数据");
writeData(new MongoStorage(), "用户数据");

Python 示例

from typing import Protocol


class DataStorage(Protocol):
    def save(self, data: str) -> None:
        ...


class MySQLStorage:
    def save(self, data: str) -> None:
        print(f"MySQL 存储: {data}")


class MongoStorage:
    def save(self, data: str) -> None:
        print(f"Mongo 存储: {data}")


def write_data(storage: DataStorage, data: str) -> None:
    storage.save(data)


write_data(MySQLStorage(), "用户数据")
write_data(MongoStorage(), "用户数据")
语言差异：TypeScript 有原生 interface 关键字，强制编译时检查；Python 用 Protocol 达到类似效果，但鸭子类型让你可以完全不写接口也能跑。设计模式中，接口是最常用的抽象手段。

六个概念的关系

概念关系关键词用在哪
封装基础能力隐藏细节，暴露接口几乎每个类
继承复用手段"是一种"模板方法、适配器等
多态灵活切换"同方法，不同行为"策略、工厂、观察者等
组合替代继承"有一个"桥接、装饰器、代理等
接口契约"定义能做什么，不说怎么做"几乎所有设计模式

今日总结

五个概念不是孤立的。真正的面向对象设计是在"该继承还是该组合"、"该暴露接口还是该封装"之间做选择。掌握这些概念后，你就能理解设计模式为什么是那些结构，而不是硬套模板。
