面向对象基础教程

面向对象的五个核心概念——封装、继承、多态、组合、接口——是所有设计模式的基石。掌握它们之间的区别和选择原则，能让你在设计模式的学习中事半功倍。

前置基础 OOP Basics TypeScript + Python 封装·继承·多态·组合·接口

今日目标

  • 理解封装、继承、多态、组合、接口的核心含义。
  • 能用 TypeScript 和 Python 写出每种概念的示例。
  • 知道继承和组合各自的适用场景。
  • 理解为什么接口是设计模式中最重要的概念之一。

1. 封装

核心思想：把数据和操作封装在对象内部，外部只能通过公开方法访问，避免直接操作内部状态。

生活类比： ATM 机只暴露插卡口和出钞口，内部电路和线缆被外壳封住。你不需要知道钱是怎么从钱箱里数出来的。

坏代码

class BankAccount {
  balance = 0;
}

const account = new BankAccount();
account.balance = -99999;  // 直接改，没有任何校验

TypeScript 示例

class BankAccount {
  private _balance = 0;

  get balance(): number {
    return this._balance;
  }

  deposit(amount: number): void {
    if (amount <= 0) throw new Error("金额必须大于 0");
    this._balance += amount;
  }

  withdraw(amount: number): void {
    if (amount <= 0) throw new Error("金额必须大于 0");
    if (amount > this._balance) throw new Error("余额不足");
    this._balance -= amount;
  }
}

Python 示例

class BankAccount:
    def __init__(self):
        self._balance = 0  # 前置下划线约定为"内部使用"

    @property
    def balance(self) -> int:
        return self._balance

    def deposit(self, amount: int) -> None:
        if amount <= 0:
            raise ValueError("金额必须大于 0")
        self._balance += amount

    def withdraw(self, amount: int) -> None:
        if amount <= 0:
            raise ValueError("金额必须大于 0")
        if amount > self._balance:
            raise ValueError("余额不足")
        self._balance -= amount
语言差异：TypeScript 有 private 关键字强制限制访问；Python 依靠命名约定（_field）和 @property，由开发者自觉遵守。

2. 继承

核心思想：子类可以从父类获取属性和方法，实现代码复用，表达"是一种"的关系。

生活类比：电动车和燃油车都"是一种"车，都有方向盘、刹车、加速踏板，但具体动力来源不同。

坏代码

class Car {
  start() { /* ... */ }
  stop() { /* ... */ }
}
class ElectricCar {
  start() { /* 重复 Car 的逻辑 */ }
  stop() { /* 重复 Car 的逻辑 */ }
}

TypeScript 示例

class Car {
  start(): void { console.log("启动"); }
  stop(): void { console.log("停止"); }
}

class ElectricCar extends Car {
  charge(): void { console.log("充电中"); }
}

const car = new ElectricCar();
car.start();
car.charge();

Python 示例

class Car:
    def start(self) -> None:
        print("启动")

    def stop(self) -> None:
        print("停止")


class ElectricCar(Car):
    def charge(self) -> None:
        print("充电中")


car = ElectricCar()
car.start()
car.charge()
继承的陷阱：过度使用继承会导致类层次过深、耦合过紧。如果子类只是"有一个"而不是"是一种"，应该用组合。

3. 多态

核心思想：同一接口，不同实现。调用方只关心"能做什么"，不关心"具体是谁做的"。

TypeScript 示例

interface Animal {
  speak(): void;
}

class Dog implements Animal {
  speak(): void { console.log("汪汪"); }
}

class Cat implements Animal {
  speak(): void { console.log("喵喵"); }
}

function makeSound(animal: Animal): void {
  animal.speak();
}

makeSound(new Dog());
makeSound(new Cat());

Python 示例

from typing import Protocol


class Animal(Protocol):
    def speak(self) -> None:
        ...


class Dog:
    def speak(self) -> None:
        print("汪汪")


class Cat:
    def speak(self) -> None:
        print("喵喵")


def make_sound(animal: Animal) -> None:
    animal.speak()


make_sound(Dog())
make_sound(Cat())
Python 的鸭子类型让多态更自然：不需要显式继承，只要有同名方法即可。

4. 组合

核心思想："有一个"比"是一种"更灵活。把对象作为属性组合起来，而不是继承。

TypeScript 示例

class Engine {
  start(): void { console.log("引擎启动"); }
}

class Car {
  constructor(private engine: Engine) {}
  start(): void { this.engine.start(); }
}

const car = new Car(new Engine());
car.start();

Python 示例

class Engine:
    def start(self) -> None:
        print("引擎启动")


class Car:
    def __init__(self, engine: Engine) -> None:
        self.engine = engine

    def start(self) -> None:
        self.engine.start()


car = Car(Engine())
car.start()
组合优于继承：组合让对象之间的关系更松散，运行时更容易替换部件。

5. 接口

核心思想：定义"能做什么"的契约，不约束"怎么做"。

TypeScript 示例

interface PaymentMethod {
  pay(amount: number): void;
}

class Alipay implements PaymentMethod {
  pay(amount: number): void { console.log(`支付宝支付 ${amount}`); }
}

class WechatPay implements PaymentMethod {
  pay(amount: number): void { console.log(`微信支付 ${amount}`); }
}

Python 示例

from typing import Protocol


class PaymentMethod(Protocol):
    def pay(self, amount: int) -> None:
        ...


class Alipay:
    def pay(self, amount: int) -> None:
        print(f"支付宝支付 {amount}")


class WechatPay:
    def pay(self, amount: int) -> None:
        print(f"微信支付 {amount}")

今日总结

封装保护数据、继承复用代码、多态统一接口、组合灵活组装、接口定义契约。设计模式本质上是这些基础概念在不同场景下的组合应用。
