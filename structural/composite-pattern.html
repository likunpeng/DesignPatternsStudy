组合模式教程

组合模式解决的是“树形结构里，单个对象和对象集合能否用同一种方式处理”的问题。它让调用方可以统一对待叶子节点和容器节点。

结构型模式Composite PatternTypeScript + Python树形结构

今日目标

  • 理解组合模式为什么适合菜单、目录、组织架构。
  • 掌握叶子节点和组合节点的共同接口。
  • 能实现递归计算。
  • 知道透明式和安全式组合的取舍。

1. 这个模式解决什么问题

树形结构中，有的节点是叶子，有的节点包含子节点。如果调用方到处判断 isFolder 或 isFile，逻辑会重复。组合模式让它们实现同一接口。

2. 一个生活类比

公司组织架构里，员工是叶子节点，部门是组合节点。你可以问一个员工“工资是多少”，也可以问一个部门“总工资是多少”。部门会把所有下级员工和子部门的工资加起来。

3. 适用场景

  • 文件目录、菜单、评论树、组织架构、权限树。
  • 调用方希望统一处理单个对象和对象集合。
  • 操作天然具有递归特征，例如统计、渲染、搜索。
  • 树结构层级可能不断变化。

4. 不适用场景

  • 数据不是树形结构。
  • 叶子和容器差异很大，强行统一接口会很别扭。
  • 需要频繁跨层级复杂查询，可能更适合图或索引结构。

5. 核心结构

角色职责
Component叶子和组合节点共同接口。
Leaf没有子节点，直接完成操作。
Composite包含子节点，把操作递归分发给子节点。
Client面向 Component 编程。

6. TypeScript 示例

interface FileNode {
  getSize(): number;
}

class FileItem implements FileNode {
  constructor(private size: number) {}
  getSize(): number { return this.size; }
}

class Folder implements FileNode {
  private children: FileNode[] = [];

  add(child: FileNode): void { this.children.push(child); }

  getSize(): number {
    return this.children.reduce((total, child) => total + child.getSize(), 0);
  }
}

const root = new Folder();
root.add(new FileItem(10));
const images = new Folder();
images.add(new FileItem(20));
images.add(new FileItem(30));
root.add(images);
console.log(root.getSize());

7. Python 示例

from typing import Protocol


class FileNode(Protocol):
    def get_size(self) -> int:
        ...


class FileItem:
    def __init__(self, size: int) -> None:
        self.size = size

    def get_size(self) -> int:
        return self.size


class Folder:
    def __init__(self) -> None:
        self.children: list[FileNode] = []

    def add(self, child: FileNode) -> None:
        self.children.append(child)

    def get_size(self) -> int:
        return sum(child.get_size() for child in self.children)


root = Folder()
root.add(FileItem(10))
images = Folder()
images.add(FileItem(20))
images.add(FileItem(30))
root.add(images)
print(root.get_size())

8. 优缺点

优点缺点
统一处理叶子和组合节点。接口过于统一时，叶子可能暴露不该有的方法。
递归逻辑清晰，便于扩展节点类型。树很深时要考虑性能和栈深度。
调用方不需要频繁判断节点类型。限制某些节点类型的子节点规则会更复杂。

9. 和相似模式的区别

模式重点区别
组合树形整体和部分一致处理关注节点层级关系。
装饰器一层层增强对象也是递归结构，但重点是增强链，不是树。
迭代器遍历集合可与组合模式配合，但不负责树结构建模。

10. 练习题

设计一个菜单系统：菜单项可以点击，菜单组包含多个菜单项或子菜单组。请用组合模式实现 render()。

参考思路：定义 MenuNode.render()，MenuItem 渲染自己，MenuGroup 遍历子节点并拼接结果。

今日总结

组合模式的核心是“部分和整体使用同一接口”。只要你的问题是树形结构，并且需要递归处理，就值得考虑它。
