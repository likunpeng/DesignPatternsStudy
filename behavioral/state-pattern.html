状态模式教程

状态模式解决的是"对象的行为取决于它的状态，而且状态会动态变化"的问题。它把状态相关的行为封装成独立的状态类，消除大量条件分支。

1. 这个模式解决什么问题

当对象需要根据内部状态改变行为时，如果用大量 if-else 或 switch 来处理，代码会变得臃肿且难以维护。比如：

  • 订单状态：待支付 → 已支付 → 已发货 → 已完成
  • 文档状态：草稿 → 审核中 → 已发布
  • 播放器状态：暂停 → 播放 → 缓冲中

每个状态下的行为不同，但对象本身是同一个。

2. 坏代码示例

class Order {
  handle() {
    if (this.state === "pending") {
      console.log("待支付，处理中...");
      this.state = "paid";
    } else if (this.state === "paid") {
      console.log("已支付，准备发货...");
      this.state = "shipped";
    } else if (this.state === "shipped") {
      console.log("已发货，交易完成");
      this.state = "completed";
    }
  }
}

问题：状态多时条件分支爆炸，每次增删状态都要修改这个方法。

3. 一个生活类比

水有三种状态：冰、水、水蒸气。不同状态下水分子的行为不同（固定不动、流动、蒸发）。但都是 H₂O。

4. 适用场景

  • 对象行为受状态影响，且状态会动态变化。
  • 有大量条件分支处理不同状态。
  • 需要消除大量 if-else 或 switch。
  • 状态转换逻辑复杂，需要封装。

5. 不适用场景

  • 状态很少，if-else 已经足够简单。
  • 状态不会动态变化，或者没有复杂的状态转换。

6. 核心结构

角色职责
Context持有状态对象，委托状态处理行为。
State定义状态接口。
ConcreteState实现具体状态下的行为，决定状态转换。

7. 示例代码

TypeScript

interface State {
  handle(context: Order): void;
}

class PendingState implements State {
  handle(context: Order) {
    console.log("待支付");
    context.setState(new PaidState());
  }
}

class PaidState implements State {
  handle(context: Order) {
    console.log("已支付");
    context.setState(new ShippedState());
  }
}

class ShippedState implements State {
  handle(context: Order) {
    console.log("已发货");
  }
}

class Order {
  private state: State;

  constructor() {
    this.state = new PendingState();
  }

  setState(state: State) { this.state = state; }
  handle() { this.state.handle(this); }
}

Python

from abc import ABC, abstractmethod

class State(ABC):
    @abstractmethod
    def handle(self, context): pass

class PendingState(State):
    def handle(self, context):
        print("待支付")
        context.set_state(PaidState())

class PaidState(State):
    def handle(self, context):
        print("已支付")
        context.set_state(ShippedState())

class ShippedState(State):
    def handle(self, context):
        print("已发货")

class Order:
    def __init__(self):
        self.state = PendingState()

    def set_state(self, state):
        self.state = state

    def handle(self):
        self.state.handle(self)

8. 优缺点

优点缺点
消除 if-else 和 switch类数量增加
状态行为封装到独立类过度使用可能使代码复杂
符合开闭原则，易于增删状态状态转换逻辑分散在各 State 类中，难以全局把控

9. 和相似模式的区别

模式区别
策略模式策略模式由客户端选择算法；状态模式由对象内部状态决定行为，状态转换是自动的。
模板方法模板方法定义算法骨架；状态模式定义不同状态下的行为。

10. 练习题

场景：实现一个播放器，支持暂停、播放、缓冲三种状态，每种状态下操作按钮的行为不同。写出代码结构。

参考答案：定义 Player 类作为 Context，定义 PlayingState、PausedState、BufferingState 三个状态类。每个状态类实现 handle 方法，决定用户按下播放/暂停按钮时的行为，并可以触发状态转换。

今日总结

状态模式通过把每种状态的行为封装到独立的状态类，让对象在状态变化时自动切换行为。它消除了大量条件分支，让状态转换逻辑更清晰，适合状态多且会动态变化的场景。
