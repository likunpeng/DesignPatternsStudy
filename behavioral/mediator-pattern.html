中介者模式教程

中介者模式解决的是"多个对象之间相互引用，导致耦合严重"的问题。它通过引入一个中介者对象，让对象之间不再直接通信，而是通过中介者协调。

行为型模式 Mediator Pattern TypeScript + Python 对象解耦

今日目标

  • 理解中介者模式解决的问题。
  • 知道什么时候应该引入中介者。
  • 能用 TypeScript 实现中介者模式。
  • 能用 Python 实现中介者模式。
  • 知道中介者模式的适用和不适用场景。

1. 这个模式解决什么问题

当多个对象相互引用时，比如对话框中的按钮、输入框、下拉框互相依赖，修改一个可能影响其他所有。引入中介者后，所有对象只和中介者通信，对象之间完全解耦。

2. 坏代码示例

class Button {
  constructor() {
    this.dialog = null;
  }
  click() {
    if (this.dialog.input.value) {
      this.dialog.list.add(this.dialog.input.value);
      this.dialog.input.clear();
    }
  }
}

class Input {
  constructor() {
    this.dialog = null;
  }
  clear() { /* ... */ }
}

class List {
  constructor() {
    this.dialog = null;
  }
  add(item) { /* ... */ }
}

问题：Button、Input、List 都直接引用 dialog，而且互相引用，耦合严重。

3. 一个生活类比

机场塔台：飞机之间不直接通信，而是通过塔台协调。飞行员只需要和塔台联系，不知道其他飞机在哪。塔台知道所有飞机的状态，负责协调起降。

4. 适用场景

  • 多个对象通过引用关系形成网状结构，耦合严重。
  • 修改一个对象会影响很多其他对象。
  • 需要复用某个类，但它的依赖很难复用。
  • GUI 组件之间的交互协调。

5. 不适用场景

  • 对象之间关系简单，几乎没有耦合。
  • 引入中介者后，中介者本身变成新的耦合中心（上帝对象）。

6. 核心结构

角色职责
Mediator定义组件之间通信的接口。
ConcreteMediator实现协调逻辑，知道所有组件。
Colleague每个组件，只和中介者通信。

7. 示例代码

TypeScript

interface Mediator {
  notify(sender: object, event: string): void;
}

class Dialog implements Mediator {
  public button: Button;
  public input: Input;

  constructor() {
    this.button = new Button(this);
    this.input = new Input(this);
  }

  notify(sender: object, event: string): void {
    if (sender === this.button && event === "click") {
      this.input.clear();
    }
  }
}

class Button {
  constructor(private mediator: Mediator) {}
  click() {
    this.mediator.notify(this, "click");
  }
}

class Input {
  constructor(private mediator: Mediator) {}
  value = "test";
  clear() {
    this.value = "";
    console.log("输入框已清空");
  }
}

Python

class Mediator:
    def notify(self, sender, event): pass

class Dialog(Mediator):
    def __init__(self):
        self.button = Button(self)
        self.input = Input(self)

    def notify(self, sender, event):
        if sender == self.button and event == "click":
            self.input.clear()

class Button:
    def __init__(self, mediator):
        self.mediator = mediator

    def click(self):
        self.mediator.notify(self, "click")

class Input:
    def __init__(self, mediator):
        self.mediator = mediator
        self.value = "test"

    def clear(self):
        self.value = ""
        print("输入框已清空")

8. 优缺点

优点缺点
消除对象之间的直接引用中介者可能变成上帝对象
减少子类生成中介者承担了太多责任
简化对象协议集中化使中介者难以维护和测试

9. 和相似模式的区别

模式区别
观察者模式观察者是单向通信；中介者是双向协调。
外观模式外观模式封装子系统；中介者模式协调组件通信。

10. 练习题

场景：实现一个聊天室，多个用户可以发送消息。不用中介者的话，每个用户都要知道其他所有用户。写出使用中介者的设计方案。

参考答案：定义 ChatRoom 作为中介者，User 作为组件。用户发送消息时调用 chatroom.send(user, message)，中介者收到后广播给其他用户。每个用户只认识聊天室，不需要知道其他用户。

今日总结

中介者模式通过引入一个协调者，让对象之间不再直接通信，从而消除对象之间的紧耦合。它适合对象之间引用关系复杂、修改互相影响的场景。
