迭代器模式教程

迭代器模式解决的是"想遍历一个集合，但不想暴露集合内部结构"的问题。它提供一种统一的方式访问集合中的元素，不管集合是数组、链表还是树。

今日目标

  • 理解迭代器模式解决的问题。
  • 知道迭代器和集合解耦的意义。

  • 能用 Python 实现迭代器模式。
  • 知道迭代器模式的适用和不适用场景。

1. 这个模式解决什么问题

遍历数组可以直接用下标，但链表、树、图等数据结构遍历方式不同。如果每次遍历都要知道集合内部结构，代码就会和集合紧耦合。

迭代器模式把"怎么遍历"封装到迭代器里，使用者只需要调用 next() 就能遍历任何集合。

2. 坏代码示例

for (let i = 0; i < array.length; i++) {
  console.log(array[i]);
}

for (let i = 0; i < linkedList.size; i++) {
  console.log(linkedList.get(i));
}

问题：遍历方式和集合类型紧耦合，换一种集合就要改遍历代码。

3. 一个生活类比

图书馆的索书系统：你不需要知道书在哪个书架、怎么排列，只需要告诉管理员要借什么书，他帮你找到。不管书在哪儿，你只和管理员（迭代器）打交道。

4. 适用场景

  • 需要统一的方式遍历不同类型的集合。
  • 不想暴露集合内部结构。
  • 需要支持多种遍历方式（前序、后序、层序等）。
  • 遍历时需要记录当前位置。

5. 不适用场景

  • 集合很简单，只需要简单 for 循环。
  • 不需要在遍历过程中修改集合。

6. 核心结构

角色职责
Iterator定义访问和遍历的接口（next, hasNext）。
ConcreteIterator实现遍历逻辑，记录当前位置。
Aggregate定义创建迭代器的接口。
ConcreteAggregate返回具体的迭代器。

7. 示例代码

TypeScript

interface Iterator<T> {
  next(): T | null;
  hasNext(): boolean;
}

class ArrayIterator<T> implements Iterator<T> {
  private index = 0;
  constructor(private collection: T[]) {}

  next(): T | null {
    return this.hasNext() ? this.collection[this.index++] : null;
  }

  hasNext(): boolean {
    return this.index < this.collection.length;
  }
}

// 真正的链表节点
class ListNode<T> {
  constructor(
    public value: T,
    public next: ListNode<T> | null = null
  ) {}
}

// 链表迭代器：遍历真实链表
class LinkedListIterator<T> implements Iterator<T> {
  private current: ListNode<T> | null;
  constructor(head: ListNode<T> | null) {
    this.current = head;
  }

  next(): T | null {
    if (!this.current) return null;
    const value = this.current.value;
    this.current = this.current.next;
    return value;
  }

  hasNext(): boolean {
    return this.current !== null;
  }
}

class LinkedList<T> {
  private head: ListNode<T> | null = null;

  add(value: T): void {
    const node = new ListNode(value);
    if (!this.head) { this.head = node; return; }
    let curr = this.head;
    while (curr.next) curr = curr.next;
    curr.next = node;
  }

  createIterator(): Iterator<T> {
    return new LinkedListIterator(this.head);
  }
}

const list = new LinkedList<number>();
list.add(1);
list.add(2);
list.add(3);
const iterator = list.createIterator();
while (iterator.hasNext()) {
  console.log(iterator.next());
}

Python

from __future__ import annotations
from typing import Any, Optional

class Iterator:
    def __iter__(self):
        return self

    def __next__(self):
        raise StopIteration

class ArrayIterator(Iterator):
    def __init__(self, collection):
        self.collection = collection
        self.index = 0

    def __next__(self):
        if self.index < len(self.collection):
            item = self.collection[self.index]
            self.index += 1
            return item
        raise StopIteration

# 真正的链表节点
class ListNode:
    def __init__(self, value, next_node=None):
        self.value = value
        self.next = next_node

class LinkedListIterator(Iterator):
    def __init__(self, head: Optional[ListNode]):
        self.current = head

    def __next__(self):
        if self.current is None:
            raise StopIteration
        value = self.current.value
        self.current = self.current.next
        return value

class LinkedList:
    def __init__(self):
        self.head = None

    def add(self, value):
        node = ListNode(value)
        if not self.head:
            self.head = node
            return
        curr = self.head
        while curr.next:
            curr = curr.next
        curr.next = node

    def __iter__(self):
        return LinkedListIterator(self.head)

ll = LinkedList()
ll.add(1)
ll.add(2)
ll.add(3)
for item in ll:
    print(item)

8. 优缺点

优点缺点
集合和遍历逻辑解耦如果只需要简单遍历，可能过度设计
支持多种遍历方式迭代器消耗额外内存
符合单一职责原则对简单数组/列表来说增加了复杂度

9. 和相似模式的区别

模式区别
组合模式组合模式处理树形结构；迭代器模式处理集合遍历。
访问者模式访问者模式对集合元素操作；迭代器模式只负责遍历。

10. 练习题

场景：实现一个二叉树的层序遍历迭代器，支持前序、中序、后序三种遍历方式。写出代码结构。

参考答案：定义 TreeIterator 类接收根节点，在构造函数中根据遍历类型初始化队列或栈。next() 方法从队列或栈中取出节点并处理其子节点。hasNext() 检查队列或栈是否为空。

今日总结

迭代器模式把集合的遍历逻辑封装到独立的迭代器对象中，让使用者不需要知道集合内部结构就能遍历元素。它是很多编程语言内置的遍历机制的基础。
