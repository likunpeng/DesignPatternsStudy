解释器模式教程

解释器模式解决的是"需要解释一种特定语言或表达式"的问题。它把语法规则表示成抽象语法树，通过遍历树来解释表达式。

1. 这个模式解决什么问题

当你需要解释一种简单语言时，比如正则表达式、数学公式、配置文件等。把语法规则用类表示，形成抽象语法树（AST），然后通过遍历树来解释执行。

2. 坏代码示例

问题：把所有语法规则混在一起，扩展困难，难以维护。

3. 一个生活类比

计算器：输入 "2 + 3"，计算器内部先把表达式解析成语法树（数字2、数字3、加号），然后从叶子到根计算。数学表达式的求值过程就是解释器的过程。

4. 适用场景

  • 需要解释一种简单语言。
  • 语法规则比较简单，不需要复杂的编译器优化。
  • 需要重复解释表达式。
  • 配置文件的解析、简单公式计算、正则表达式等。

5. 不适用场景

  • 语法规则复杂，此时应该用 Parser Generator。
  • 性能要求高，需要更高效的解析技术。
  • 只需要简单计算，直接 eval 更省事。

6. 核心结构

角色职责
AbstractExpression声明抽象解释方法。
TerminalExpression终结符表达式，如数字、变量。
NonterminalExpression非终结符表达式，如加法、乘法。
Context包含全局信息或当前状态。

7. 示例代码

TypeScript

interface Expression {
  interpret(context: Record<string, number>): number;
}

class NumberExpression implements Expression {
  constructor(private num: number) {}
  interpret(): number { return this.num; }
}

class VariableExpression implements Expression {
  constructor(private name: string) {}
  interpret(context: Record<string, number>): number {
    return context[this.name] ?? 0;
  }
}

class AddExpression implements Expression {
  constructor(private left: Expression, private right: Expression) {}
  interpret(context: Record<string, number>): number {
    return this.left.interpret(context) + this.right.interpret(context);
  }
}

class MultiplyExpression implements Expression {
  constructor(private left: Expression, private right: Expression) {}
  interpret(context: Record<string, number>): number {
    return this.left.interpret(context) * this.right.interpret(context);
  }
}

// 解析: a + 2 * b, 其中 a=3, b=5
const expr = new AddExpression(
  new VariableExpression("a"),
  new MultiplyExpression(
    new NumberExpression(2),
    new VariableExpression("b")
  )
);
console.log(expr.interpret({ a: 3, b: 5 })); // 3 + 2*5 = 13

Python

class Expression:
    def interpret(self, context):
        pass

class NumberExpression(Expression):
    def __init__(self, num):
        self.num = num

    def interpret(self, context):
        return self.num

class VariableExpression(Expression):
    def __init__(self, name):
        self.name = name

    def interpret(self, context):
        return context.get(self.name, 0)

class AddExpression(Expression):
    def __init__(self, left, right):
        self.left = left
        self.right = right

    def interpret(self, context):
        return self.left.interpret(context) + self.right.interpret(context)

class MultiplyExpression(Expression):
    def __init__(self, left, right):
        self.left = left
        self.right = right

    def interpret(self, context):
        return self.left.interpret(context) * self.right.interpret(context)


# 解析: a + 2 * b, 其中 a=3, b=5
expr = AddExpression(
    VariableExpression("a"),
    MultiplyExpression(
        NumberExpression(2),
        VariableExpression("b")
    )
)
print(expr.interpret({"a": 3, "b": 5}))  # 3 + 2*5 = 13

8. 优缺点

优点缺点
容易改变和扩展语法规则语法规则复杂时类数量爆炸
易于实现简单语法不适合复杂语法，性能较低
符合开闭原则维护复杂语法树困难

9. 和相似模式的区别

模式区别
访问者模式访问者模式对已有结构操作；解释器模式定义语言的语法和语义。
组合模式解释器模式用组合模式构建 AST。

10. 练习题

场景：实现一个布尔表达式解释器，支持 AND、OR、NOT 操作和变量。如：a AND (b OR c)。写出代码结构。

参考答案：定义 BooleanExpression 接口，AndExpression、OrExpression、NotExpression、VariableExpression 实现它。interpret(context) 方法递归解释表达式，context 存储变量布尔值。

今日总结

解释器模式通过把语法规则表示成类，构建抽象语法树来解释表达式。它适合简单语言的解释场景，但语法复杂时应该用专门的解析器生成器。
