备忘录模式教程

备忘录模式解决的是"需要保存对象的某个状态，以便后面恢复"的问题。它在不破坏封装性的前提下，捕获对象内部状态并保存起来。

1. 这个模式解决什么问题

撤销操作需要保存对象之前的状态。如果直接访问对象内部状态，会破坏封装性。备忘录模式把状态封装成独立对象，由 Caretaker（管理者）保存。

2. 坏代码示例

class Editor {
  content: string;
  cursor: number;
  fontSize: number;

  save() {
    return {
      content: this.content,
      cursor: this.cursor,
      fontSize: this.fontSize
    };
  }

  restore(state) {
    this.content = state.content;
    this.cursor = state.cursor;
    this.fontSize = state.fontSize;
  }
}

问题：Editor 的内部状态暴露了，任何人可以修改它。

3. 一个生活类比

游戏存档：玩到某一关时保存进度（备忘录），之后如果那关没过，可以读档（恢复状态）。玩家不需要知道存档文件里具体存了什么。

4. 适用场景

  • 需要保存对象状态以便撤销。
  • 不能破坏对象的封装性。
  • 需要实现快照或回滚功能。
  • 事务支持，需要回滚。

5. 不适用场景

  • 状态很小，简单变量就能保存。
  • 不需要撤销/恢复功能。
  • 状态可以接受被暴露。

6. 核心结构

角色职责
Memento存储原发器内部状态的快照。
Originator创建备忘录，指定保存哪些状态；根据备忘录恢复状态。
Caretaker保存和管理备忘录，不操作状态内容。

7. 示例代码

TypeScript

class Memento {
  constructor(private state: string) {}
  getState(): string { return this.state; }
}

class Editor {
  private content = "";

  type(words: string) { this.content += words; }
  getContent(): string { return this.content; }

  save(): Memento { return new Memento(this.content); }
  restore(memento: Memento) { this.content = memento.getState(); }
}

class History {
  private mementos: Memento[] = [];

  push(m: Memento) { this.mementos.push(m); }
  pop(): Memento | undefined { return this.mementos.pop(); }
}

const editor = new Editor();
const history = new History();

editor.type("Hello");
history.push(editor.save());

editor.type(" World");
console.log(editor.getContent());

const memento = history.pop();
if (memento) {
  editor.restore(memento);
}
console.log(editor.getContent());

Python

class Memento:
    def __init__(self, state):
        self._state = state

    def get_state(self):
        return self._state

class Editor:
    def __init__(self):
        self._content = ""

    def type(self, words):
        self._content += words

    def get_content(self):
        return self._content

    def save(self):
        return Memento(self._content)

    def restore(self, memento):
        if memento is None:
            return
        self._content = memento.get_state()

class History:
    def __init__(self):
        self._mementos = []

    def push(self, m):
        self._mementos.append(m)

    def pop(self):
        return self._mementos.pop() if self._mementos else None

editor = Editor()
history = History()

editor.type("Hello")
history.push(editor.save())

editor.type(" World")
print(editor.get_content())

editor.restore(history.pop())
print(editor.get_content())

8. 优缺点

优点缺点
保存对象状态不破坏封装性消耗内存，频繁保存状态可能占用大量资源
简化 Originator 代码某些语言不支持确定性内存管理
支持撤销/恢复Caretaker 需要管理大量快照，版本控制复杂

9. 和相似模式的区别

模式区别
命令模式命令模式保存操作；备忘录模式保存状态。
迭代器模式迭代器模式遍历集合；备忘录模式保存状态快照。

10. 练习题

场景：实现一个图形编辑器，支持画线、画圆、撤销、重做。每个图形有自己的位置、大小、颜色属性。写出代码结构。

参考答案：定义 Shape 作为 Originator，每个图形可以 save() 返回 Memento。Caretaker（History）保存多个 Memento 实现重做。undo 时恢复上一个 Memento，redo 时恢复下一个 Memento。

今日总结

备忘录模式在不破坏封装性的前提下，捕获对象内部状态并保存。它是实现撤销/恢复、快照、回滚等功能的基础，适合需要保存和恢复对象状态的场景。
