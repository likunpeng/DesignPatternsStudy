模板方法模式教程

模板方法模式解决的是"多个类有相同的流程，但每一步的具体实现不同"的问题。它把不变的整体流程抽象到父类，把可变的细节留给子类。

行为型模式 Template Method Pattern TypeScript + Python 流程抽象

今日目标

  • 理解模板方法模式解决的问题。
  • 知道什么时候应该把公共流程提取到父类。
  • 能用 TypeScript 实现模板方法模式。
  • 能用 Python 实现模板方法模式。
  • 知道模板方法模式的适用和不适用场景。

1. 这个模式解决什么问题

多个类需要执行相同的操作步骤，但每一步的具体实现不同。比如：

  • 泡茶和泡咖啡的流程都是：烧水 → 泡 → 倒入杯中 → 加调料
  • 编译 Java 和编译 Python 的流程都是：词法分析 → 语法分析 → 生成代码
  • 电商下单和线下购买的流程都是：验证 → 扣款 → 发货

如果每个类都重复写相同的流程代码，就会造成大量重复。而且当流程需要修改时，要改很多地方。

2. 坏代码示例

假设泡茶和泡咖啡各写一套代码：

class Tea {
  make() {
    this.boilWater();
    this.steep();
    this.pourInCup();
    this.addLemon();
  }
  // ... 其他方法
}

class Coffee {
  make() {
    this.boilWater();
    this.brew();
    this.pourInCup();
    this.addMilk();
  }
  // ... 其他方法
}

问题：流程 烧水 → 泡 → 倒入杯中 → 加调料 重复了 2 次，如果需要修改流程（比如增加"清洗杯子"步骤），要改 2 个地方。

3. 一个生活类比

做早餐的流程：打开冰箱 → 取出食材 → 烹饪 → 装盘。不管是做煎蛋还是做三明治，这个流程都一样，变的只是每个步骤的具体操作。有一个"早餐制作模板"，可以套用到所有早餐上。

4. 适用场景

  • 多个类有相同的操作步骤。
  • 步骤顺序重要，不能随意调换。
  • 不变的部分应该集中管理，可变的部分由子类实现。
  • 需要控制子类的扩展方式（通过钩子方法）。

5. 不适用场景

  • 各个类的操作步骤完全不同，没有公共流程。
  • 想要使用继承但 JavaScript/TypeScript 的类继承机制受限。
  • 只需要组合，不需要继承层次结构。

6. 核心结构

角色职责
AbstractClass定义模板方法和骨架步骤，调用抽象方法或钩子方法。
ConcreteClass实现各个步骤的具体逻辑。
Template Method定义算法骨架的方法，一般不允许重写。
Primitive Operation子类必须实现的抽象方法。
Hook可选的钩子方法，子类可以选择重写。

7. 示例代码

TypeScript

abstract class CaffeineBeverage {
  prepare() {
    this.boilWater();
    this.brew();
    this.pourInCup();
    if (this.customerWantsCondiments()) {
      this.addCondiments();
    }
  }

  abstract brew(): void;
  abstract addCondiments(): void;

  boilWater() { console.log("烧水"); }
  pourInCup() { console.log("倒入杯中"); }

  customerWantsCondiments(): boolean { return true; }
}

class Coffee extends CaffeineBeverage {
  brew() { console.log("冲泡咖啡"); }
  addCondiments() { console.log("加入牛奶"); }
  customerWantsCondiments(): boolean { return confirm("要加牛奶吗？"); }
}

class Tea extends CaffeineBeverage {
  brew() { console.log("浸泡茶叶"); }
  addCondiments() { console.log("加入柠檬"); }
}

Python

from abc import ABC, abstractmethod

class CaffeineBeverage(ABC):
    def prepare(self):
        self.boil_water()
        self.brew()
        self.pour_in_cup()
        if self.customer_wants_condiments():
            self.add_condiments()

    @abstractmethod
    def brew(self): pass

    @abstractmethod
    def add_condiments(self): pass

    def boil_water(self): print("烧水")
    def pour_in_cup(self): print("倒入杯中")
    def customer_wants_condiments(self): return True

class Coffee(CaffeineBeverage):
    def brew(self): print("冲泡咖啡")
    def add_condiments(self): print("加入牛奶")

class Tea(CaffeineBeverage):
    def brew(self): print("浸泡茶叶")
    def add_condiments(self): print("加入柠檬")

8. 优缺点

优点缺点
消除重复代码，把公共流程集中到父类类数量增加，每个具体步骤都要一个类
符合开闭原则，易于扩展新子类模板方法默认不允许重写，灵活性受限
通过钩子方法可以让子类控制算法步骤继承层次过深可能导致代码难理解

9. 和相似模式的区别

模式区别
策略模式策略模式把不同的算法封装成独立类，可以互相替换；模板方法通过继承实现，算法步骤固定。
工厂方法工厂方法专注于对象创建；模板方法专注于算法流程。

10. 练习题

场景：实现一个数据导出功能，支持导出为 CSV 和 JSON 两种格式。两种格式的流程都是：获取数据 → 格式化 → 写入文件。写出代码结构。

参考答案：定义抽象类 DataExporter，包含模板方法 export()，调用抽象方法 fetchData()、format()、writeToFile()。然后创建 CsvExporter 和 JsonExporter 两个子类实现具体逻辑。

今日总结

模板方法模式通过继承机制，把不变的整体流程提取到父类，把可变的细节留给子类实现。它遵循"复用相同逻辑，个性化不同细节"的思想，适合流程固定但步骤多变的场景。
