模板方法模式教程

模板方法模式解决的是"多个类有相同的流程，但每一步的具体实现不同"的问题。它把不变的整体流程抽象到父类，把可变的细节留给子类。

行为型模式 Template Method Pattern TypeScript + Python 流程抽象

今日目标

  • 理解模板方法模式解决的问题。
  • 知道什么时候应该把公共流程提取到父类。

  • 能用 Python 实现模板方法模式。

1. 这个模式解决什么问题

多个类需要执行相同的操作步骤，但某些步骤的具体实现不同。比如：

  • CSV 和 JSON 导出的流程都是：获取数据 → 校验数据 → 格式化 → 写文件 → 记录日志
  • 编译 Java 和编译 Python 的流程都是：词法分析 → 语法分析 → 生成代码
  • 电商下单和线下购买的流程都是：验证 → 扣款 → 发货

如果每个类都重复写相同的流程代码，就会造成大量重复。而且当流程需要修改时，要改很多地方。

2. 坏代码示例

假设 CSV 和 JSON 导出各写一套代码：

type Row = { name: string; age: number };

class CsvExporter {
  export() {
    // CSV 导出流程：获取数据、校验、格式化、写文件、记录日志。
    const data = this.fetchData();
    if (data.length === 0) {
      throw new Error("没有可导出的数据");
    }

    const content = this.formatCsv(data);
    this.writeFile(content);
    console.log("CSV 导出完成");
  }

  fetchData(): Row[] {
    return [{ name: "Tom", age: 18 }];
  }

  formatCsv(data: Row[]): string {
    return data.map(item => `${item.name},${item.age}`).join("\n");
  }

  writeFile(content: string) {
    console.log("写入文件:", content);
  }
}

class JsonExporter {
  export() {
    // JSON 导出流程和 CSV 几乎一样，只有格式化方式不同。
    const data = this.fetchData();
    if (data.length === 0) {
      throw new Error("没有可导出的数据");
    }

    const content = this.formatJson(data);
    this.writeFile(content);
    console.log("JSON 导出完成");
  }

  fetchData(): Row[] {
    return [{ name: "Tom", age: 18 }];
  }

  formatJson(data: Row[]): string {
    return JSON.stringify(data);
  }

  writeFile(content: string) {
    console.log("写入文件:", content);
  }
}

问题：流程 获取数据 → 校验数据 → 格式化 → 写文件 → 记录日志 重复了 2 次。如果以后所有导出都要增加"权限检查"或"性能埋点"，每个导出类都要修改，而且很容易漏改。

3. 一个生活类比

体检中心的流程通常是固定的：登记 → 检查项目 → 出报告 → 归档。不同套餐的检查项目不同，但整体顺序不能乱。模板方法就像体检中心规定好的流程表，具体套餐只填入自己那部分检查项目。

4. 适用场景

  • 多个类有相同的操作步骤。
  • 步骤顺序重要，不能随意调换。
  • 不变的部分应该集中管理，可变的部分由子类实现。
  • 需要控制子类的扩展方式（通过钩子方法）。

5. 不适用场景

  • 各个类的操作步骤完全不同，没有公共流程。
  • 想要使用继承但 JavaScript/TypeScript 的类继承机制受限。
  • 只需要组合，不需要继承层次结构。

6. 核心结构

角色职责
AbstractClass定义模板方法和骨架步骤，调用抽象方法或钩子方法。
ConcreteClass实现各个步骤的具体逻辑。
Template Method定义算法骨架的方法，一般不允许重写。
Primitive Operation子类必须实现的抽象方法。
Hook可选的钩子方法，子类可以选择重写。

7. 示例代码

TypeScript

type Row = { name: string; age: number };

// 抽象父类：定义所有导出的固定流程。
abstract class DataExporter {
  export() {
    // 模板方法：规定导出骨架，子类不能随意改变步骤顺序。
    const data = this.fetchData();
    this.validate(data);
    const content = this.format(data);
    this.writeFile(content);
    this.log();
  }

  // 公共步骤：默认从同一个数据源获取数据。
  protected fetchData(): Row[] {
    return [{ name: "Tom", age: 18 }];
  }

  // 公共步骤：所有导出都复用同一套校验规则。
  protected validate(data: Row[]) {
    if (data.length === 0) {
      throw new Error("没有可导出的数据");
    }
  }

  // 抽象步骤：不同导出格式必须自己实现。
  protected abstract format(data: Row[]): string;

  // 公共步骤：所有导出都写入文件。
  protected writeFile(content: string) {
    console.log("写入文件:", content);
  }

  // 钩子方法：子类可以重写日志内容，不重写也有默认行为。
  protected log() {
    console.log("导出完成");
  }
}

// CSV 子类：只实现 CSV 格式化这个变化点。
class CsvExporter extends DataExporter {
  protected format(data: Row[]): string {
    return data.map(item => `${item.name},${item.age}`).join("\n");
  }
}

// JSON 子类：只实现 JSON 格式化这个变化点。
class JsonExporter extends DataExporter {
  protected format(data: Row[]): string {
    return JSON.stringify(data);
  }
}

Python

import json
from abc import ABC, abstractmethod

Row = dict[str, object]


# 抽象父类：定义所有导出的固定流程。
class DataExporter(ABC):
    def export(self):
        # 模板方法：统一规定导出步骤。
        data = self.fetch_data()
        self.validate(data)
        content = self.format(data)
        self.write_file(content)
        self.log()

    # 公共步骤：默认从同一个数据源获取数据。
    def fetch_data(self) -> list[Row]:
        return [{"name": "Tom", "age": 18}]

    # 公共步骤：所有导出都复用同一套校验规则。
    def validate(self, data: list[Row]) -> None:
        if not data:
            raise ValueError("没有可导出的数据")

    # 抽象步骤：不同导出格式必须自己实现。
    @abstractmethod
    def format(self, data: list[Row]) -> str:
        pass

    # 公共步骤：所有导出都写入文件。
    def write_file(self, content: str) -> None:
        print("写入文件:", content)

    # 钩子方法：子类可以重写日志内容，不重写也有默认行为。
    def log(self) -> None:
        print("导出完成")


# CSV 子类：只实现 CSV 格式化这个变化点。
class CsvExporter(DataExporter):
    def format(self, data: list[Row]) -> str:
        return "\n".join(f"{item['name']},{item['age']}" for item in data)


# JSON 子类：只实现 JSON 格式化这个变化点。
class JsonExporter(DataExporter):
    def format(self, data: list[Row]) -> str:
        return json.dumps(data, ensure_ascii=False)

8. 优缺点

优点缺点
消除重复代码，把公共流程集中到父类类数量增加，每个具体步骤都要一个类
符合开闭原则，易于扩展新子类模板方法默认不允许重写，灵活性受限
通过钩子方法可以让子类控制算法步骤继承层次过深可能导致代码难理解

9. 和相似模式的区别

模式区别
策略模式策略模式把不同的算法封装成独立类，可以互相替换；模板方法通过继承实现，算法步骤固定。
工厂方法工厂方法专注于对象创建；模板方法专注于算法流程。

10. 练习题

场景：实现一个文件导入功能，支持导入 CSV 和 JSON 两种格式。两种格式的流程都是：读取文件 → 校验内容 → 解析数据 → 保存数据 → 记录日志。写出代码结构。

参考答案：定义抽象类 FileImporter，包含模板方法 import()，调用公共方法 readFile()、validate()、save()、log()，把 parse() 设计成抽象方法。然后创建 CsvImporter 和 JsonImporter 两个子类实现不同解析逻辑。

今日总结

模板方法模式通过继承机制，把不变的整体流程提取到父类，把可变的细节留给子类实现。它遵循"复用相同逻辑，个性化不同细节"的思想，适合流程固定但步骤多变的场景。
