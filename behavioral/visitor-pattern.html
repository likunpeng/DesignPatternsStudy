访问者模式教程

访问者模式解决的是"需要对一个对象结构执行多种操作，但不想在元素类中添加这些操作"的问题。它把操作封装到访问者类中，元素类只负责接受访问者。

行为型模式 Visitor Pattern TypeScript + Python 操作分离

今日目标

  • 理解访问者模式解决的问题。
  • 知道什么时候应该使用访问者模式。
  • 能用 TypeScript 实现访问者模式。
  • 能用 Python 实现访问者模式。
  • 知道访问者模式的适用和不适用场景。

1. 这个模式解决什么问题

当你需要对一组相似对象（如文件、文件夹）执行多种操作（如计算大小、杀毒、压缩）时，如果把每种操作都加到对象类里，会导致类膨胀且难以维护。访问者模式把操作封装到访问者中，对象只负责"接受访问"。

2. 坏代码示例

class File {
  calculateSize() { /* ... */ }
  scan() { /* ... */ }
  compress() { /* ... */ }
}

class Folder {
  calculateSize() { /* ... */ }
  scan() { /* ... */ }
  compress() { /* ... */ }
}

问题：每新增一种操作都要修改 File 和 Folder。操作和对象紧耦合。

3. 一个生活类比

博物馆的解说系统：参观者（访问者）来到博物馆，博物馆（对象结构）负责开放各个展区。参观者可以是学生、记者、老年人等（不同访问者），他们从同一展区获取不同信息。博物馆不需要知道参观者是谁。

4. 适用场景

  • 对象结构稳定，但需要对其执行多种操作。
  • 需要对一个对象结构执行很多不同操作。
  • 操作需要访问对象内部的属性（访问者持有元素引用）。
  • 编译器、AST 处理、文件遍历等场景。

5. 不适用场景

  • 对象结构经常变化，不稳定。
  • 新增操作会破坏元素类的封装。
  • 只需要简单遍历，不需要复杂操作。

6. 核心结构

角色职责
Visitor为每种元素定义 visit 操作。
ConcreteVisitor实现具体的访问操作。
Element定义 accept 方法，接受访问者。
ObjectStructure管理元素集合。

7. 示例代码

TypeScript

interface Visitor {
  visitFile(file: File): void;
  visitFolder(folder: Folder): void;
}

interface Node {
  accept(visitor: Visitor): void;
}

class File implements Node {
  constructor(public name: string, public size: number) {}
  accept(visitor: Visitor) { visitor.visitFile(this); }
}

class Folder implements Node {
  children: Node[] = [];
  constructor(public name: string) {}
  add(node: Node) { this.children.push(node); }
  accept(visitor: Visitor) { visitor.visitFolder(this); }
}

class SizeCalculator implements Visitor {
  private total = 0;
  visitFile(file: File) { this.total += file.size; }
  visitFolder(folder: Folder) {
    folder.children.forEach(c => c.accept(this));
  }
  getTotal(): number { return this.total; }
  reset(): void { this.total = 0; }
}

const folder = new Folder("root");
folder.add(new File("a.txt", 100));
folder.add(new File("b.txt", 200));

const calculator = new SizeCalculator();
folder.accept(calculator);
console.log(calculator.getTotal()); // 300

// 复用同一个计算器前先重置
calculator.reset();
const folder2 = new Folder("docs");
folder2.add(new File("c.txt", 50));
folder2.accept(calculator);
console.log(calculator.getTotal()); // 50

Python

class Visitor:
    def visit_file(self, file): pass
    def visit_folder(self, folder): pass

class Node:
    def accept(self, visitor): pass

class File(Node):
    def __init__(self, name, size):
        self.name = name
        self.size = size

    def accept(self, visitor):
        visitor.visit_file(self)

class Folder(Node):
    def __init__(self, name):
        self.name = name
        self.children = []

    def add(self, node):
        self.children.append(node)

    def accept(self, visitor):
        visitor.visit_folder(self)

class SizeCalculator(Visitor):
    def __init__(self):
        self.total = 0

    def visit_file(self, file):
        self.total += file.size

    def visit_folder(self, folder):
        for c in folder.children:
            c.accept(self)

    def reset(self):
        self.total = 0

folder = Folder("root")
folder.add(File("a.txt", 100))
folder.add(File("b.txt", 200))

calculator = SizeCalculator()
folder.accept(calculator)
print(calculator.total)  # 300

# 复用同一个计算器前先重置
calculator.reset()
folder2 = Folder("docs")
folder2.add(File("c.txt", 50))
folder2.accept(calculator)
print(calculator.total)  # 50

8. 优缺点

优点缺点
符合开闭原则，新增操作不改元素类元素类需要暴露内部状态给访问者
操作集中管理，易于维护对象结构变化时，所有访问者都要改
操作可以累加状态破坏元素封装，难以添加新元素类型

9. 和相似模式的区别

模式区别
迭代器模式迭代器模式遍历元素；访问者模式对元素执行操作。
装饰器模式装饰器模式添加功能；访问者模式添加操作。

10. 练习题

场景：实现一个编译器中的 AST（抽象语法树），支持表达式求值（VisitorA）和代码生成（VisitorB）。写出代码结构。

参考答案：定义 NumberNode、AddNode、MultiplyNode 等作为 Element，实现 accept 方法。定义 Evaluator 和 CodeGenerator 作为 Visitor。ObjectStructure 是 AST 根节点，遍历子节点调用 accept。

今日总结

访问者模式把对一组元素的操作封装到访问者类中，让元素负责接受访问者。它适合对象结构稳定但操作经常变化的场景，如编译器、报表生成、文件处理等。
